# 🛡️ Update Threat Models with New Data Inputs

> **Continuously Improve Security Posture Through Dynamic Threat Modeling, MITRE ATT&CK Integration, Telemetry Analysis, and Automated Threat Intelligence**

---

## 📖 Overview

Modern products continuously generate telemetry, logs, network events, and security findings. Static threat models quickly become outdated as attackers evolve their techniques and infrastructures change.

This repository provides a hands-on lab for building **dynamic threat models** that automatically evolve using new telemetry and network intelligence. Students will learn how to ingest multiple security data sources, analyze attack patterns, map findings to the **MITRE ATT&CK Framework**, generate actionable threat intelligence reports, visualize attack trends, and automate continuous threat model updates using Python.

Instead of manually updating security documentation, this lab demonstrates how organizations can create living threat models that remain relevant throughout the product lifecycle.

---

# 🎯 Repository Purpose

This repository teaches students how to build an automated threat intelligence pipeline capable of:

- 🔍 Collecting telemetry and network security data

- 🎯 Mapping attacks to MITRE ATT&CK techniques

The lab combines **Threat Modeling**, **MITRE ATT&CK**, **Security Analytics**, **Automation**, and **Threat Intelligence** into one complete workflow.
